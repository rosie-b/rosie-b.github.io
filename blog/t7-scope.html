#15 Scope

What is scope and how does it work?

Scope in JavaScript is related to where we declare variables and functions, which impacts on accessibility of these throughout the source code. Scope can either be global or local.

Global
This is where we declare a variable outside of any particular function. This variable is then accessible/visible globally to all functions throughout the source code.

This could be helpful if your source code is relatively simple/short and you know that a variable will be used across a few functions. You can easily update details for this variable, without the need to go into the individual functions to do this (reducing the risk of human error or introducing bugs in the process).

That said, it is a good idea to keep global scope to a minimum, to reduce the risk of accidentally using a global variable within a function (e.g. spelling mistake) and the ramifications this could have on performance, output or otherwise. Also, if you consider that when writing a code you are trying to break this down into simple modules and functions with one purpose, if you were to have many functions linking up to the same global variables, this could get very complicated and make for a lot of dependency issues.

Local
When a variable is declared within a function, this then is only accessible/visible locally within the function (or in functions nested further within this one).

The use of local scope is helpful when dealing with large, complex source code (i.e. breaking down into modular compnents) and enables the use of the same variable names across multiple functions, without overwriting or impacting on eachother.
