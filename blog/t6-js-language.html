#12 JavaScript Language

Parentheses, brackets, braces, quotes - what are they all used for?

In the JavaScript language parentheses, brackets, braces and quotes are used to do things such as store information, invoke functions and define data types.

  • parentheses () are used for grouping and or containing expressions. This can be helpful when wanting to dictate how a sum is going to be calculated.
    e.g. var a = (4 + 1) * 5
  • Also used in functions for storing parameters. For instance, if you want to do a count of animals, but only those that are cats.
    e.g. function countAnimals(cat)
  • And when making 'if' statements, you would use the parentheses to contain the condition that you were looking to check against.
    e.g. function countAnimals() { if ( animal = "cat" ) { return count; } }
  • brackets [] are used to store data (i.e. arrays of information), which could be a simple list attached to one variable. Or it could be a list of data held within a property of an object.
    e.g. var animals = [ "cat", dog", "mouse" ]
  • Brackets are then also used for accessing data in the arrays, but inputting the index number of where that piece of information sits within the list.
    e.g. console.log ( animals [1] );
    returns "dog"
  • Note: The index in an array starts at the number 0. Therefore, in the above example, "cat" is at position 0, "dog" is at position 1 and "mouse" is at position 2.
  • braces {} are used to declare a piece of data as being an object. Within the braces, you then record all the properties of that object.
    e.g. var ball = { shape: round, colour: red, size: big }
  • Braces are also used to enclose blocks of code, such as a series of conditions within a formula or a for loop.
  • single quotes '' and double quote " " are used when writing strings (e.g. a thing or a sentence, rather than a number or true/false). Both types of quotes are used for the same things and can be used as you wish, so long as the start and end of the string match (i.e. either ' ' OR " ").
  • You can use a double quote within a string that has the start/end of a single quote, without exiting the string itself (and vice versa). This is useful when writing pieces of text that require the output to show "quotes".
  • When declaring a variable which you know is going to be in the form of a string, but that you are yet to assign a value to, you can can assign " " to it and this will tell the variable to expect a string.
    e.g. var animal = " ";
